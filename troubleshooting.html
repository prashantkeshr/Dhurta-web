Troubleshooting

Something not working right?

Most issues here come from a security toggle interacting with a specific site, not a broken install. Start with the category closest to your problem.

Privacy toggles VPN & Connect Extensions & media Gestures & performance Data & resets

Privacy toggles

A video site won't play, or throws a playback error, after I turned on Anti-Fingerprint

Anti-Fingerprint adds deliberate noise to canvas, audio and WebGL reads to make your browser harder to identify. Some sites — video platforms in particular — run an integrity check against canvas output, and noisy canvas data can trip that check.

Turn Anti-Fingerprint off in the Security panel for that session, reload the tab, and the playback should return to normal. This is a real trade-off of the feature, not a bug — it's why the toggle defaults to off.

Security panel
The warmth / display panel doesn't seem to open as a dropdown from the URL bar

Web page content in Dhurta renders in a layer that sits above the rest of the interface. To keep controls clickable, some panels — like the download tray and warmth slider — are built to open inline within the URL bar's own height instead of dropping down over the page.

If a panel looks clipped, try widening the window slightly, or look for the control expanding within the bar itself rather than below it.

Interface behavior
Ghost Mode tabs still show up in my regular history

They shouldn't — Ghost tabs use a separate in-memory session and are never written to the local SQLite history. If you're seeing entries that look like a Ghost session in your Data Hub, confirm the tab was actually opened as Ghost Tab (right-click → Ghost Tab) and not a regular tab in a private-looking state.

VPN & Connect

A site still shows my old region after I switched VPN countries

The proxy applies to all sessions as soon as you change the country selector, but an already-loaded page may be holding onto a connection it opened before the switch. Hard-refresh the tab (Ctrl+Shift+R) or open a new tab to confirm the new route.

Security panel — VPN
My Connect pairing code expired before the other device entered it

Pairing codes are short-lived by design, so a stale code can't be reused later. Reopen Connect on the first device to generate a fresh code, then enter it promptly on the second.

Connect
Connect isn't finding the other Dhurta instance

Both windows need to be on and actively showing the Connect panel at the same time — it isn't a background listener. Double-check the code was typed exactly as shown, including leading digits.

Extensions & media

"Load unpacked" fails or shows nothing after selecting a folder

Dhurta expects the folder you select to directly contain a manifest.json — not a parent folder with the extension nested a level deeper. Point the file picker at the exact folder with manifest.json inside it.

Extensions panel
Picture-in-Picture doesn't appear on a video I'm hovering

The floating PiP button only injects on standard HTML5 <video> elements. Some sites use custom players that block the native video controls — for those, use the PiP button in the URL bar instead, which falls back to popping out the whole page.

Picture-in-Picture
A page's popup window opened as a new tab instead

That's intentional — Dhurta intercepts popup windows and opens them as tabs so they can't hide outside the sidebar or spawn always-on-top ad windows.

Interface behavior

Gestures & performance

Two-finger swipe back/forward isn't registering on my trackpad

Check your OS trackpad settings first — some drivers intercept horizontal two-finger swipes for their own workspace-switching gesture before Dhurta ever sees the event. Disabling that OS-level gesture usually resolves it.

Gestures
Pinch-to-zoom feels jumpy on a page

Zoom is clamped between 25% and 500% per page. If a page defines its own touch/pinch handling (some canvas-heavy apps do), it can compete with Dhurta's zoom gesture — try the Ctrl + scroll or the zoom buttons in the URL bar instead.

Gestures

Data & resets

After wiping my data, all my security toggles reset to off

That's expected. A full data wipe deletes the local database entirely, and every setting — including Anti-Fingerprint, WebRTC blocking, and VPN — reseeds to its default, which is off. Re-enable whichever toggles you want from the Security panel after a wipe or reinstall.

Settings — Privacy
My wallpaper disappeared after moving my picture library

Wallpapers are stored as embedded data, not a link to the original file, so moving or renaming the source image shouldn't affect it. If it did disappear, re-select it from Settings → Appearance and it will be re-embedded.

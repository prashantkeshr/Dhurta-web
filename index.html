How it actually works

Four layers. Zero compromise.

Every session in Dhurta passes through independently verifiable protection layers — not marketing claims, not opt-ins. Defaults that protect.

Ghost Mode — Real Tor Onion Routing

Traffic exits through the Tor network — not a proxy, not a simulation. Verify it yourself at check.torproject.org from inside a Ghost tab. Session lives in memory; nothing written to disk.

Verifiable

Chakra Shield — Bundle Mode

One switch enables VPN + Anti-Fingerprint + WebRTC Block simultaneously. No partial activation. If VPN fails to connect, Dhurta warns you instead of silently falling back.

Atomic

Anti-Fingerprint + WebRTC Block

Canvas, AudioContext, and font metrics return spoofed noise — making cross-site fingerprint tracking statistically incoherent. WebRTC is fully disabled so your real IP cannot leak through media APIs.

Always-on

Local-Only SQLite — Zero Telemetry

All your data — history, bookmarks, settings, extensions — lives in a single SQLite file on your machine. Nothing is synced to Dhurta servers because there are none. Check the Transparency Dashboard to see every byte stored about you.
